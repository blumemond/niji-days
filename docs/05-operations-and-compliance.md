# 運用・権利・データ方針

## 非公式表記

本サービスは非公式ファンツールとして公開します。

フッターやAbout画面に次の内容を明記します。

- 本サービスは非公式ファンツールです
- ANYCOLOR株式会社およびにじさんじ公式とは関係ありません
- 掲載情報は正確性に注意していますが、誤りが含まれる可能性があります
- データ修正依頼の連絡先

## 権利・素材方針

MVPでは次の方針を採用します。

- 公式ロゴは使わない
- 公式画像、ライバー画像、サムネイルを無断転載しない
- 名前、日付、公式リンク、カラーを中心に構成する
- 見た目を公式サイトに過度に寄せない
- サービス名やドメインで公式と誤認される表現を避ける

リリース前に確認する公式情報:

- [ANYCOLOR二次創作ガイドライン](https://www.anycolor.co.jp/guidelines/)
- [にじさんじ公式サイト](https://www.nijisanji.jp/)

## デビュー日の基準

初期案では「初配信日」をデビュー日として扱います。

ただし、ライバーによって公式発表日、初配信日、初投稿日、活動開始日の扱いが揺れる可能性があります。そのため、各ライバーに `debut_date_basis` と `source_note` を持たせます。

表示例:

```text
このサイトでは初配信日をデビュー日として扱っています。
データに誤りがある場合は修正依頼フォームからお知らせください。
```

## データ更新フロー

MVPでは手動更新を前提とします。

1. 公式プロフィール、公式告知、初配信アーカイブなどから候補日を確認する
2. `data/livers.json` を更新する
3. 日付計算テストを実行する
4. 変更差分をレビューする
5. 公開する

将来的には管理画面やGitHub Pull Requestによる修正受付を検討します。

## データ品質ルール

- デビュー日は必ずISO形式 `YYYY-MM-DD` で保存する
- タイムゾーンは `Asia/Tokyo` を基準にする
- 情報源メモを残す
- 不確かな情報は `source_note` に明記する
- 修正履歴をGitで追えるようにする

## 問い合わせ・修正依頼

修正依頼フォームでは、根拠URLの入力を推奨します。

分類:

- デビュー日が違う
- 表示名が違う
- 所属・グループが違う
- リンク切れ
- その他

## リスク

### 公式誤認リスク

対策:

- 非公式表記を目立つ場所に置く
- 公式ロゴや公式サイト風のUIを避ける
- サービス名を独自名称にする

### データ誤りリスク

対策:

- デビュー日の基準を明示する
- 根拠メモを持つ
- 修正報告導線を用意する

### ファン間トラブルリスク

対策:

- ランキング風の見せ方を避ける
- 記念日を祝う文脈に絞る
- 人気順、登録数順などの比較指標を出さない
