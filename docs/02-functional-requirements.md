# 機能要件

## 優先度

- P0: MVPに必須
- P1: 初期公開後すぐ欲しい
- P2: 将来拡張

## P0 機能

### トップダッシュボード

トップページでは、ファンが最短で状況を把握できるように次のブロックを表示します。

- 今日の記念日
- 推し一覧
- 近日記念日
- 全ライバー検索への導線

表示ルール:

- 今日ちょうど節目のライバーを最上部に表示する
- ユーザーが推し登録している場合、推し一覧を今日の記念日の次に表示する
- 推し登録がない場合、推し登録を促す小さな空状態を表示する
- 近日記念日は標準で30日以内を表示する

### ライバー一覧

全ライバーを検索・絞り込みできる一覧画面。

表示項目:

- ライバー名
- デビュー日
- 経過日数
- 次の記念日
- 所属区分
- 活動状態
- 推し登録ボタン

検索・並び替え:

- 名前検索
- デビュー日が新しい順
- デビュー日が古い順
- 経過日数が多い順
- 記念日が近い順
- 推し登録済みのみ

フィルター:

- JP / EN / その他の区分
- デビュー期、同期、グループ
- 現役 / 卒業済み / 全て

### ライバー詳細

ライバーごとの詳細ページ。

表示項目:

- ライバー名
- 今日の経過日数
- デビュー日
- デビュー日の基準
- 次の記念日までの残り日数
- 近い節目一覧
- 公式リンク
- 推し登録ボタン
- X共有文生成ボタン

### 記念日計算

初期対応する節目:

- 100日ごと
- 500日ごと
- 1000日ごと
- 1周年以降の周年
- 半周年

計算仕様:

- 日数はサービスの基準タイムゾーンである `Asia/Tokyo` の日付単位で計算する
- デビュー日当日は `1日目` と表示する
- 経過日数の内部値は `targetDate - debutDate + 1` とする
- 「デビューから何日経過」はユーザー表示上 `N日目` とする
- 「あと何日」は `anniversaryDate - today` とする

表記例:

- 今日で `1,234日目`
- `1,250日まであと16日`
- `5周年まであと102日`

### 推し登録

ログイン不要で、ブラウザのローカルストレージに保存します。

仕様:

- ライバーごとに推し登録 / 解除できる
- トップページに推しだけを表示する
- 推し登録情報は端末・ブラウザ単位
- ブラウザ保存であることを設定画面に明記する

### X共有文生成

ライバー詳細ページから共有文を生成します。

共有文例:

```text
今日は〇〇さんのデビューから1,234日目です。
次の1,250日記念まであと16日。

#NijiDays
```

仕様:

- ユーザーが投稿前に編集できるよう、Xの投稿画面へテキスト付きURLで遷移する
- ハッシュタグは初期値として入れるが、ユーザーが編集可能
- 公式タグの利用は運用方針で検討し、濫用しない

## P1 機能

### カレンダー

月単位で記念日を確認できる画面。

表示:

- 今日のマーカー
- 節目がある日
- ライバー名
- 節目ラベル

### データ修正報告

ユーザーがデータ誤りを報告できるフォーム。

最低項目:

- 対象ライバー
- 修正したい項目
- 正しいと思う値
- 根拠URL
- 任意コメント

### 表示設定

ユーザーが表示を調整できる設定。

- 近日記念日の範囲: 7日 / 14日 / 30日 / 60日
- 卒業済みライバーの表示
- 日数表記: `N日目` / `デビューからN日`
- テーマ: ライト / ダーク / OS連動

## P2 機能

- 通知機能
- Google Calendar / iCal エクスポート
- 共有画像生成
- 誕生日や3Dお披露目日などのイベント拡張
- 多言語対応
- 管理画面
- ユーザーアカウント同期

## 非機能要件

### パフォーマンス

- トップページの初期表示は3秒以内を目標とする
- ライバー一覧はクライアント側検索で即時反応する規模を想定する
- データは静的JSONまたは軽量APIから取得できる設計にする

### アクセシビリティ

- 色だけに依存しない状態表示にする
- ボタンには明確なラベルを付ける
- キーボードで検索、フィルター、推し登録が操作できる
- コントラスト比を意識する

### レスポンシブ

- スマホ利用を主対象とする
- 360px幅でも主要情報が崩れない
- デスクトップでは情報量を増やし、一覧比較しやすくする
